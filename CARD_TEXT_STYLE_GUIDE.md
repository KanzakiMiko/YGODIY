# 游戏王 DIY 卡文写法指南

本文档给负责制作 MDPro3 DIY 卡片的 agent 使用。目标是让卡文尽量接近官方简体中文卡库的读感，并且让 Lua 脚本、CDB 文本、游戏内显示三者表达一致。

本指南只规范「卡片效果文本怎么写」，不决定卡片平衡性，也不替代脚本实现检查。

## 1. 基本原则

写卡文时先想清楚规则逻辑，再写文字。不要先写漂亮句子，再让脚本去猜。

优先顺序：

1. 效果规则必须准确。
2. 卡文结构必须能分清发动条件、费用、对象、处理。
3. 术语尽量贴近 Konami 官方简体中文卡库。
4. 同一个扩展包内术语、数字、标点保持一致。
5. 卡文不要解释脚本实现，只描述玩家在决斗中看到和执行的效果。

项目内统一使用偏官方简中口吻：

- 用 `此卡`，不要混用 `这张卡`。
- 用 `此卡名`，不要混用 `这个卡名`。
- 用 `对手`，不要混用 `对方`。
- 用 `牌组`、`额外牌组`，不要写 `卡组`、`额外卡组`。
- 用 `墓地`、`除外状态`、`场上`、`手牌`。
- 用 `怪兽区域`、`魔法与陷阱区域`。
- 用 `特殊召唤`，不要写 `特召`。
- 用 `发动` 表示发动卡或效果，用 `使用` 表示次数限制。
- 用半角数字：`1回合`、`1张`、`1只`、`1000`。
- 编号效果用 `①：`、`②：`、`③：`。
- 并列类别用 `・`：`手牌・牌组`、`魔法・陷阱・怪兽的效果`。

不要写口语说明，例如：

```text
这张卡可以一回合用一次，把对面的怪炸了。
```

改成：

```text
此卡名的①效果1回合仅可使用1次。
①：以对手场上的1只怪兽为对象可以发动。将该怪兽破坏。
```

## 2. 卡文整体顺序

一张卡的效果文本按以下顺序组织：

```text
素材行或召唤条件
特殊召唤限制或规则限制
此卡名/此卡的次数限制
①：第1个效果。
②：第2个效果。
③：第3个效果。
```

### 2.1 额外牌组怪兽素材行

额外牌组怪兽的素材行放在最前面，不编号。

融合：

```text
「幽灵宝可梦」怪兽＋效果怪兽
```

同步：

```text
协调＋协调以外的怪兽1只以上
```

超量：

```text
等级4怪兽×2
```

连接：

```text
「幽灵宝可梦」怪兽2只
```

素材要求要短而明确。不要把召唤后效果写进素材行。

### 2.2 召唤条件和召唤限制

不能通常召唤、只能用指定方式特殊召唤、额外牌组特殊召唤限制等，通常放在编号效果前。

```text
此卡不可通常召唤，仅可通过...特殊召唤。
此卡仅可通过融合召唤方式特殊召唤。
此卡从额外牌组仅可在将自己场上的上述卡除外的情况下特殊召唤。
```

如果是「可从手牌特殊召唤」这类能发动或能适用的效果，可以编号：

```text
①：自己场上有「幽灵宝可梦」怪兽存在的情况下，此卡可从手牌特殊召唤。
```

### 2.3 次数限制放在编号效果前

多个编号效果共享卡名次数限制时，优先合并写在编号效果前。不要在每个编号效果中重复长句。

推荐：

```text
此卡名的①②效果1回合仅可各使用1次。
①：...
②：...
```

不推荐：

```text
①：此卡名的①效果1回合仅可使用1次。...
②：此卡名的②效果1回合仅可使用1次。...
```

## 3. 次数限制写法

### 3.1 卡名限制

同名卡共享次数，写 `此卡名`。

```text
此卡名的①效果1回合仅可使用1次。
此卡名的①②效果1回合仅可各使用1次。
此卡名的①②③效果1回合仅可各使用1次。
```

如果多个效果互斥：

```text
此卡名的①②效果1回合1次，仅可使用其中1个。
```

如果特殊召唤方法和效果分别限制：

```text
此卡名以①方法的特殊召唤1回合仅可进行1次，且②效果1回合仅可使用1次。
```

如果禁止同一连锁重复发动：

```text
此卡名的①②效果1回合仅可各使用1次，不可在同一连锁上发动。
```

### 3.2 非卡名限制

同一张表侧存在的卡每回合能用一次，写 `1回合1次`，不要写 `此卡名`。

```text
②：1回合1次，在自己的主要阶段可以发动。从手牌将1只...特殊召唤。
```

如果双方回合各一次：

```text
③：在每个自己・对手回合中1次，...可以发动。
```

### 3.3 魔法・陷阱卡本身的发动限制

魔法・陷阱卡「这张卡本身」1回合只能发动1张，写：

```text
此卡名的卡1回合仅可发动1张。
```

如果还有墓地效果等编号效果限制：

```text
此卡名的卡1回合仅可发动1张，此卡名的②效果1回合仅可使用1次。
```

注意：

- `发动1张` 用于魔法・陷阱卡本身的发动。
- `使用1次` 用于效果次数限制。
- 永续魔法・永续陷阱在场上已有的效果，通常仍用 `效果1回合仅可使用1次`。

## 4. 效果句法：条件、费用、对象、处理

先把每个效果拆成四个部分：

```text
①：发动条件，费用，以对象为对象可以发动。效果处理。
```

不是每个效果都有四部分，但顺序不要乱。

### 4.1 发动条件

发动条件描述「什么时候可以发动」。

常见写法：

```text
此卡召唤・特殊召唤的情况下可以发动。
此卡被送至墓地的情况下可以发动。
自己场上有「幽灵宝可梦」怪兽存在的情况下可以发动。
在自己的主要阶段可以发动。
对手发动魔法・陷阱・怪兽的效果时可以发动。
```

可选效果写 `可以发动`。

必发效果不写 `可以发动`：

```text
②：此卡与怪兽进行战斗的伤害计算后发动。将该怪兽破坏。
```

`的情况下` 和 `时` 不要随便替换。一般规则：

- `的情况下可以发动` 更宽松，常用于现代卡文。
- `时可以发动` 常用于需要直接响应某个发动或事件的效果。
- 对连锁响应类效果通常写 `...时可以发动`。

### 4.2 费用

费用在发动时支付，写在 `可以发动` 前。

常见费用：

```text
支付500LP可以发动。
舍弃1张手牌可以发动。
将手牌的此卡送至墓地可以发动。
从墓地将此卡除外可以发动。
将自己场上的1只怪兽解放可以发动。
去除此卡的1个超量素材可以发动。
```

有费用又有对象时，费用在前，对象在后：

```text
舍弃1张手牌，以场上的1张卡为对象可以发动。将该卡破坏。
```

不要把费用写到处理里：

```text
不推荐：①：可以发动。舍弃1张手牌，将场上的1张卡破坏。
推荐：①：舍弃1张手牌，以场上的1张卡为对象可以发动。将该卡破坏。
```

### 4.3 对象

取对象的效果必须在 `可以发动` 前明确写 `以...为对象`。

```text
以场上的1只怪兽为对象可以发动。将该怪兽破坏。
以自己墓地的1只「幽灵宝可梦」怪兽为对象可以发动。将该怪兽特殊召唤。
以对手墓地的最多2张卡为对象可以发动。将该卡除外。
```

处理时用 `该卡`、`该怪兽`、`该对象怪兽` 指回对象。

如果处理时仍要求对象满足条件，可以写得更严格：

```text
以场上的1只表侧表示怪兽为对象可以发动。将该表侧表示怪兽的效果无效。
```

如果是不取对象的选择，把选择写在处理部分：

```text
①：可以发动。挑选对手场上的1张卡破坏。
②：可以发动。将对手场上的怪兽全部破坏。
```

不要把不取对象效果写成取对象。

### 4.4 处理

处理写在 `可以发动。` 后面。

常见处理：

```text
从牌组将1只「幽灵宝可梦」怪兽加入手牌。
从手牌将此卡特殊召唤。
将该卡破坏。
将该怪兽除外。
自己抽1张。
给予对手1000伤害。
此卡的攻击力上升1000。
```

需要持续到回合结束：

```text
此卡的攻击力直至回合结束时为止上升1000。
此回合，自己从额外牌组仅可特殊召唤不死族怪兽。
```

以此效果特殊召唤的后续限制：

```text
以此效果特殊召唤的怪兽的效果无效化。
以此效果特殊召唤的怪兽将在结束阶段破坏。
以此效果特殊召唤的怪兽离开场上的情况下除外。
```

## 5. 发动、适用、不入连锁

不是所有文本都要写 `可以发动`。

### 5.1 入连锁效果

会发动的效果，通常写编号和 `可以发动` 或 `发动`。

```text
①：此卡召唤・特殊召唤的情况下可以发动。从牌组将1张...加入手牌。
```

### 5.2 永续效果

永续效果不写 `可以发动`。

```text
①：只要此卡存在于怪兽区域，自己场上的「幽灵宝可梦」怪兽的攻击力上升500。
②：场上的此卡不会因对手的效果被破坏。
```

### 5.3 召唤规则或特殊召唤手续

不入连锁的特殊召唤通常写 `可从手牌特殊召唤`，不写 `发动`。

```text
①：自己场上有「幽灵宝可梦」怪兽存在的情况下，此卡可从手牌特殊召唤。
```

如果它在脚本中是起动效果或诱发效果，则写 `可以发动。从手牌将此卡特殊召唤。`

卡文必须和脚本实现一致。

## 6. 连接词与处理顺序

复杂处理不要随便用连接词。连接词会影响「是否同时发生」「前一处理失败后是否继续」以及「最后发生了什么」。

简化原则：

- 能拆成两句就拆成两句。
- 必须表达先后关系时，用 `然后`。
- 必须表达同时或附加关系时，先找官方近似卡文，再决定用词。
- 不确定时，不要自己发明 `并且`、`再`、`接着`。

常用写法：

```text
将该卡破坏。然后，自己抽1张。
将该发动无效并破坏。
从牌组将1张...加入手牌。此回合，自己不可...
```

PSCT 中常见英文连接词大意：

- `then`：A 先发生，B 后发生；A 没发生则不继续 B。
- `also`：A 和 B 同时处理，尽量处理能处理的部分。
- `and if you do`：A 和 B 同时看待，但 B 依赖 A。
- `and`：A 和 B 绑定，通常必须都能处理。

中文 DIY 卡文不要强行逐字翻译这些英文词。优先参考官方简中已有类似效果。

## 7. 无效、破坏、连锁限制

### 7.1 无效发动

用于无效「卡或效果的发动」：

```text
对手发动魔法・陷阱・怪兽的效果时可以发动。将该发动无效并破坏。
```

如果不破坏：

```text
将该发动无效。
```

### 7.2 无效效果

用于效果已经发动，只无效效果：

```text
将该效果无效。
```

### 7.3 无效表侧怪兽效果

用于持续性地让场上怪兽无效：

```text
以场上的1只表侧表示怪兽为对象可以发动。将该表侧表示怪兽的效果直至回合结束时为止无效。
```

### 7.4 连锁限制

```text
应对此效果的发动，对手不可发动魔法・陷阱・怪兽的效果。
```

或针对特定类别：

```text
应对此卡的发动，对手不可发动魔法・陷阱卡。
```

不要写成口语：

```text
对手不能连锁这个效果。
```

## 8. 锁定、限制与副作用

发动后的自肃：

```text
此效果发动后，直至回合结束时为止，自己不可从额外牌组特殊召唤不死族以外的怪兽。
```

卡本身发动后的自肃：

```text
此卡发动后，直至回合结束时为止，自己不可特殊召唤「幽灵宝可梦」怪兽以外的怪兽。
```

处理后的回合限制：

```text
此回合，自己不可进行战斗阶段。
此回合，自己仅可用「幽灵宝可梦」怪兽攻击。
```

特殊召唤怪兽限制：

```text
以此效果特殊召唤的怪兽不可攻击。
以此效果特殊召唤的怪兽不可作为连接素材。
以此效果特殊召唤的怪兽离开场上的情况下除外。
```

## 9. 选择效果与项目符号

二选一或多选一效果用 `●`。

```text
①：可以从以下效果中选择1个发动。
●从牌组将1只「幽灵宝可梦」怪兽加入手牌。
●以自己墓地的1只「幽灵宝可梦」怪兽为对象，将该怪兽特殊召唤。
```

如果多个选项各自有次数限制，写清楚：

```text
此卡名的以下效果1回合仅可各选择1次。
①：可以从以下效果中选择1个发动。
●...
●...
```

如果同一回合只能选一个：

```text
此卡名的①效果1回合仅可使用1次。
①：可以从以下效果中选择1个发动。
●...
●...
```

## 10. 常见卡种模板

### 10.1 主卡组怪兽

```text
此卡名的①②效果1回合仅可各使用1次。
①：此卡召唤・特殊召唤的情况下可以发动。从牌组将1张「幽灵宝可梦」魔法・陷阱卡加入手牌。
②：从墓地将此卡除外，以自己墓地的1只「幽灵宝可梦」怪兽为对象可以发动。将该怪兽特殊召唤。
```

### 10.2 融合怪兽

```text
「幽灵宝可梦」怪兽＋暗属性怪兽
此卡名的①②效果1回合仅可各使用1次。
①：此卡融合召唤的情况下可以发动。挑选场上的1张卡破坏。
②：此卡被对手破坏的情况下可以发动。从墓地将1只「幽灵宝可梦」怪兽特殊召唤。
```

### 10.3 同步怪兽

```text
协调＋协调以外的怪兽1只以上
此卡名的①②效果1回合仅可各使用1次。
①：此卡同步召唤的情况下可以发动。从牌组将1只「幽灵宝可梦」怪兽送至墓地。
②：对手发动怪兽的效果时可以发动。将该效果无效。
```

### 10.4 超量怪兽

```text
等级4「幽灵宝可梦」怪兽×2
此卡名的①②效果1回合仅可各使用1次。
①：去除此卡的1个超量素材可以发动。从牌组将1张「幽灵宝可梦」卡加入手牌。
②：此卡被破坏的情况下可以发动。将自己墓地的1只「幽灵宝可梦」怪兽特殊召唤。
```

### 10.5 连接怪兽

```text
「幽灵宝可梦」怪兽2只
此卡名的①②效果1回合仅可各使用1次。
①：此卡连接召唤的情况下，以自己墓地的1只「幽灵宝可梦」怪兽为对象可以发动。将该怪兽特殊召唤至此卡所连接区。
②：此卡所连接区的怪兽的攻击力上升500。
```

### 10.6 通常魔法

```text
此卡名的卡1回合仅可发动1张。
①：从牌组将1只「幽灵宝可梦」怪兽加入手牌。此卡发动后，直至回合结束时为止，自己不可特殊召唤「幽灵宝可梦」怪兽以外的怪兽。
```

### 10.7 速攻魔法

```text
此卡名的卡1回合仅可发动1张。
①：以自己场上的1只「幽灵宝可梦」怪兽为对象可以发动。该怪兽的攻击力直至回合结束时为止上升1000。此回合，该怪兽不会因战斗被破坏。
```

### 10.8 永续魔法

```text
此卡名的①②效果1回合仅可各使用1次。
①：作为此卡发动时的效果处理，可从牌组将1只「幽灵宝可梦」怪兽加入手牌。
②：在自己的主要阶段可以发动。从手牌将1只「幽灵宝可梦」怪兽特殊召唤。
```

### 10.9 装备魔法

```text
仅可装备给「幽灵宝可梦」怪兽。
①：装备怪兽的攻击力上升800。
②：装备怪兽被破坏的情况下，可以作为代替将此卡破坏。
```

### 10.10 场地魔法

```text
此卡名的卡1回合仅可发动1张。
①：作为此卡发动时的效果处理，可从牌组将1只「幽灵宝可梦」怪兽加入手牌。
②：自己场上的「幽灵宝可梦」怪兽的攻击力上升300。
③：1回合1次，在自己的主要阶段可以发动。从手牌将1只「幽灵宝可梦」怪兽特殊召唤。
```

### 10.11 通常陷阱

```text
此卡名的卡1回合仅可发动1张。
①：对手将怪兽特殊召唤的情况下，以场上的1只怪兽为对象可以发动。将该怪兽破坏。
```

### 10.12 反击陷阱

```text
①：对手发动魔法・陷阱・怪兽的效果时，自己场上有「幽灵宝可梦」怪兽存在的情况下可以发动。将该发动无效并破坏。
```

### 10.13 永续陷阱

```text
此卡名的①②效果1回合仅可各使用1次。
①：作为此卡发动时的效果处理，可从自己墓地将1只「幽灵宝可梦」怪兽特殊召唤。
②：自己场上的「幽灵宝可梦」怪兽被战斗・效果破坏的情况下可以发动。挑选对手场上的1张卡破坏。
```

## 11. 常见改写示例

### 11.1 重复次数限制

不规范：

```text
①：此卡名的①效果1回合只能使用1次。此卡召唤成功时可以发动...
②：此卡名的②效果1回合只能使用1次。自己场上的这张卡以外的...
```

规范：

```text
此卡名的①②效果1回合仅可各使用1次。
①：此卡召唤成功的情况下可以发动。...
②：...
```

### 11.2 费用和处理混在一起

不规范：

```text
①：可以发动。丢弃1张手牌，破坏场上1张卡。
```

规范：

```text
①：舍弃1张手牌，以场上的1张卡为对象可以发动。将该卡破坏。
```

### 11.3 不取对象写成取对象

如果脚本是处理时选择：

```text
①：此卡特殊召唤的情况下可以发动。挑选对手场上的1张卡破坏。
```

如果脚本是发动时选择对象：

```text
①：此卡特殊召唤的情况下，以对手场上的1张卡为对象可以发动。将该卡破坏。
```

卡文必须和脚本一致。

### 11.4 魔法卡发动限制

不规范：

```text
此卡名的效果1回合只能使用1次。
①：发动后，从牌组检索...
```

规范：

```text
此卡名的卡1回合仅可发动1张。
①：从牌组将1只「幽灵宝可梦」怪兽加入手牌。
```

### 11.5 口语特殊召唤

不规范：

```text
这张卡被送墓时，可以拉墓地一只本家。
```

规范：

```text
①：此卡被送至墓地的情况下，以自己墓地的1只「幽灵宝可梦」怪兽为对象可以发动。将该怪兽特殊召唤。
```

## 12. Agent 写卡文流程

其他 agent 在写入 CDB 前，必须按这个顺序处理：

1. 先用自然语言写清楚每个效果的规则逻辑。
2. 判断每个效果是否入连锁。
3. 判断是否有费用。
4. 判断是否取对象。
5. 判断处理是选择、全部处理，还是引用发动时对象。
6. 判断次数限制是卡名限制、软次数限制，还是魔法・陷阱卡发动限制。
7. 按模板写成卡文。
8. 对照脚本确认卡文和实现一致。
9. 再写入 `texts.desc`。

写入前检查清单：

```text
[ ] 是否统一使用「此卡」「此卡名」「对手」「牌组」？
[ ] 编号效果是否使用 ①：②：③：？
[ ] 共通次数限制是否放在编号效果前？
[ ] 魔法・陷阱卡本身是否写成「此卡名的卡1回合仅可发动1张」？
[ ] 每个发动效果是否能分清条件、费用、对象、处理？
[ ] 取对象效果是否写了「以...为对象可以发动」？
[ ] 不取对象效果是否没有误写对象？
[ ] 费用是否写在「可以发动」前？
[ ] 必发效果是否没有写「可以发动」？
[ ] 永续效果是否没有写成发动效果？
[ ] 自肃和后续限制是否写清持续时间？
[ ] 卡文是否和 Lua 脚本行为一致？
```

## 13. 与 Lua 脚本的对应提醒

卡文不是装饰文本，必须能对应脚本。

常见对应关系：

```text
以...为对象可以发动        -> 脚本里应有 SetTarget / SelectTarget 一类取对象逻辑
挑选...                  -> 多数情况下是处理时选择，不应在发动时取对象
舍弃/支付/解放...可以发动 -> 脚本里应作为 cost
此卡名的①效果...         -> 脚本里应有同名次数限制 count limit
此卡名的卡1回合仅可发动1张 -> 魔法・陷阱的卡发动本身需要限制
只要此卡存在...           -> 通常是永续效果，不应入连锁
```

如果脚本做不到卡文描述的精度，优先改脚本；如果 MDPro3 API 限制导致做不到，必须改卡文，不能让文字承诺不存在的规则。

## 14. 官方参考

写新卡文时优先检索官方简体中文卡库中相似效果，而不是凭感觉造句。

参考入口：

- Konami 官方卡库简体中文：`https://www.db.yugioh-card.com/yugiohdb/card_search.action?request_locale=cn`
- PSCT 条件 / 发动 / 处理结构：`https://www.yugioh-card.com/en/play/psct/psct-3/`
- PSCT 费用、对象、连锁线索：`https://www.yugioh-card.com/en/play/psct/psct-4/`
- PSCT 连接词和处理顺序：`https://www.yugioh-card.com/en/play/psct/psct-7/`

可参考的官方简中卡文类型：

- `此卡名的①②效果1回合仅可各使用1次。`
- `此卡名的卡1回合仅可发动1张。`
- `此卡名的①②效果1回合1次，仅可使用其中1个。`
- `以...为对象可以发动。将该...`
- `从手牌・牌组...`
- `应对此效果的发动，对手不可发动...`

最终原则：先找相似官方卡，再改成 DIY 卡需要的字段。不要让 agent 自创一套看起来像游戏王、但规则语义不清的中文。
