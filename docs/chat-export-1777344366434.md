# 游戏王原创卡组：《大明皇帝·承运之战》
**“日月重开，天命在吾！”**

## 一、卡组背景与设计哲学

《大明皇帝》卡组是一套以“历史推进”和“皇权更迭”为核心战术的超量召唤主题卡组。卡组的主角是明朝十六位皇帝，他们作为怪兽卡存在于主卡组与额外卡组中。魔法与陷阱卡则再现了明朝的重大历史事件与典章制度。

**卡组核心战术**：
通过下级怪兽（藩王/皇亲）检索本家核心魔法卡 **《大明律·皇明祖训》**，利用其效果不断将不同“庙号”的皇帝怪兽从卡组送入墓地，再从墓地特殊召唤，实现“皇位传承”。每次“皇位传承”（特殊召唤新的皇帝），都会触发不同的增益效果，最终通过超量召唤召唤出强大的**阶级１２“明太祖·洪武大帝”**或**阶级１３“明成祖·永乐大帝”**终结比赛。

**历史梗融合**：每张卡的效果都精准对应了该皇帝的历史功绩、性格特点或著名事件。卡组内的配合与克制关系也模仿了明朝历史的内部逻辑（如削藩、宦官专权、党争、改革等）。

## 五、卡组战术与COMBO深度解析

### 核心启动

《大明皇帝》卡组的启动核心在于**检索《大明律·皇明祖训》**。最快的启动路径是：
- 通召`001 燕王朱棣`，检索`025 皇明祖训`或`026 靖难之变`。
- 发动`皇明祖训`，检索第二只怪兽（如`002 宁王朱权`）。
- 利用`皇明祖训`的②效果，从卡组堆墓`018 永乐大帝（等级１３）`，然后从墓地苏生`017 洪武大帝（等级１２）`。注意，这个效果拉出来的怪兽效果无效，所以洪武大帝的清场效果不会触发，但它的③效果（保护“大明”怪兽不被效破）是永续效果，依然适用。

这个启动模式是对“洪武开国，永乐后继”的历史再现。朱棣（燕王）启动，堆墓朱棣（皇帝形态），拉朱元璋（太祖皇帝）站场。皇位传承从开国皇帝开始。

### 中盘战术：王朝的迭代

当`皇明祖训`在场时，每一回合都可以执行一次“送旧迎新”。通过堆墓高等级大明怪兽（通常是皇帝），然后从墓地拉另一只皇帝。
- **战术选择：** 送墓`020 宣德帝朱瞻基（R9）`，苏生`022 嘉靖帝朱厚熜（R10）`。
- 嘉靖帝出场时，可以再次发动`皇明祖训`的效果①（检索）或利用其自身效果检索`033 炼丹炉`。
- 配合`031 一条鞭法`，将场上用过的低级怪兽（如检索上来的藩王、阁臣）全部解放抽卡，再复活两只铺满场。

**历史映射**：这完美再现了明朝中期的皇帝权力运作——内阁（下级怪兽）提供支持，皇权（皇帝怪兽）在首辅（玩家）的操纵下依次更替，每换一次都有新的政令（效果）。

### 终端压制：永宣盛世

卡组的终极目标是制造出最强的终端。
第一种终端是**阶级１２的`洪武大帝`**，最好是自己正规超量召唤的（拥有素材），这样就能有一次神级清场与持续的压制。
第二种终端更灵活，是**阶级１３的`永乐大帝朱棣`**。

场上达成`永乐大帝 + 持有郑和作为素材`的场景，意味着接近完全的掌控：
- 对方回合发动`永乐大帝`②效果，手卡或墓地拉怪（如拉`015 吴三桂`，吴三桂再在对方回合跳对方的大怪）。
- 自己回合，永乐大帝宣言卡名，除外对方墓地或场上的关键卡。

达成永乐大帝的最优路径是：
- 通召`013 郑和`，检索`028 宝船`。
- `皇明祖训`堆`永乐大帝`，苏生`洪武大帝`（效果无效）。
- 发动`宝船`除外墓地多余怪兽抽２。
- 自己回合主要阶段２，发动`026 靖难之变`，解放场上某只藩王，直接从额外拉`阶级１３ 永乐大帝`。
- 下一回合，用任何方法把郑和堆下墓，再通过`永乐大帝`的自身效果把郑和拉回来，作为素材重叠进去。或者直接让郑和在场上与两只R7以上的大明怪兽组成素材进行超量召唤。

**注意**：
朱棣的效果②“手卡·墓地把１只大明怪兽特殊召唤”，这个效果是非同调·超量的通用苏生。可以直接拉墓地里的`郑和`，而`郑和`作为超量素材时赋予的穿透与抽卡能力，能每回合稳定拉开资源差。

### 残局与逆转

# 游戏王 DIY 卡文写法指南

本文档给负责制作 MDPro3 DIY 卡片的 agent 使用。目标是让卡文尽量接近官方简体中文卡库的读感，并且让 Lua 脚本、CDB 文本、游戏内显示三者表达一致。

本指南只规范「卡片效果文本怎么写」，不决定卡片平衡性，也不替代脚本实现检查。

## 1. 基本原则

写卡文时先想清楚规则逻辑，再写文字。不要先写漂亮句子，再让脚本去猜。

优先顺序：

1. 效果规则必须准确。
2. 卡文结构必须能分清发动条件、费用、对象、处理。
3. 术语尽量贴近 Konami 官方简体中文卡库。
4. 同一个扩展包内术语、数字、标点保持一致。
5. 卡文不要解释脚本实现，只描述玩家在决斗中看到和执行的效果。

项目内统一使用偏官方简中口吻：

- 用 `此卡`，不要混用 `这张卡`。
- 用 `此卡名`，不要混用 `这个卡名`。
- 用 `对手`，不要混用 `对方`。
- 用 `牌组`、`额外牌组`，不要写 `卡组`、`额外卡组`。
- 用 `墓地`、`除外状态`、`场上`、`手牌`。
- 用 `怪兽区域`、`魔法与陷阱区域`。
- 用 `特殊召唤`，不要写 `特召`。
- 用 `发动` 表示发动卡或效果，用 `使用` 表示次数限制。
- 用半角数字：`1回合`、`1张`、`1只`、`1000`。
- 编号效果用 `①：`、`②：`、`③：`。
- 并列类别用 `・`：`手牌・牌组`、`魔法・陷阱・怪兽的效果`。

不要写口语说明，例如：

```text
这张卡可以一回合用一次，把对面的怪炸了。
```

改成：

```text
此卡名的①效果1回合仅可使用1次。
①：以对手场上的1只怪兽为对象可以发动。将该怪兽破坏。
```

## 2. 卡文整体顺序

一张卡的效果文本按以下顺序组织：

```text
素材行或召唤条件
特殊召唤限制或规则限制
此卡名/此卡的次数限制
①：第1个效果。
②：第2个效果。
③：第3个效果。
```

### 2.1 额外牌组怪兽素材行

额外牌组怪兽的素材行放在最前面，不编号。

融合：

```text
「幽灵宝可梦」怪兽＋效果怪兽
```

同步：

```text
协调＋协调以外的怪兽1只以上
```

超量：

```text
等级4怪兽×2
```

连接：

```text
「幽灵宝可梦」怪兽2只
```

素材要求要短而明确。不要把召唤后效果写进素材行。

### 2.2 召唤条件和召唤限制

不能通常召唤、只能用指定方式特殊召唤、额外牌组特殊召唤限制等，通常放在编号效果前。

```text
此卡不可通常召唤，仅可通过...特殊召唤。
此卡仅可通过融合召唤方式特殊召唤。
此卡从额外牌组仅可在将自己场上的上述卡除外的情况下特殊召唤。
```

如果是「可从手牌特殊召唤」这类能发动或能适用的效果，可以编号：

```text
①：自己场上有「幽灵宝可梦」怪兽存在的情况下，此卡可从手牌特殊召唤。
```

### 2.3 次数限制放在编号效果前

多个编号效果共享卡名次数限制时，优先合并写在编号效果前。不要在每个编号效果中重复长句。

推荐：

```text
此卡名的①②效果1回合仅可各使用1次。
①：...
②：...
```

不推荐：

```text
①：此卡名的①效果1回合仅可使用1次。...
②：此卡名的②效果1回合仅可使用1次。...
```

## 3. 次数限制写法

### 3.1 卡名限制

同名卡共享次数，写 `此卡名`。

```text
此卡名的①效果1回合仅可使用1次。
此卡名的①②效果1回合仅可各使用1次。
此卡名的①②③效果1回合仅可各使用1次。
```

如果多个效果互斥：

```text
此卡名的①②效果1回合1次，仅可使用其中1个。
```

如果特殊召唤方法和效果分别限制：

```text
此卡名以①方法的特殊召唤1回合仅可进行1次，且②效果1回合仅可使用1次。
```

如果禁止同一连锁重复发动：

```text
此卡名的①②效果1回合仅可各使用1次，不可在同一连锁上发动。
```

### 3.2 非卡名限制

同一张表侧存在的卡每回合能用一次，写 `1回合1次`，不要写 `此卡名`。

```text
②：1回合1次，在自己的主要阶段可以发动。从手牌将1只...特殊召唤。
```

如果双方回合各一次：

```text
③：在每个自己・对手回合中1次，...可以发动。
```

### 3.3 魔法・陷阱卡本身的发动限制

魔法・陷阱卡「这张卡本身」1回合只能发动1张，写：

```text
此卡名的卡1回合仅可发动1张。
```

如果还有墓地效果等编号效果限制：

```text
此卡名的卡1回合仅可发动1张，此卡名的②效果1回合仅可使用1次。
```

注意：

- `发动1张` 用于魔法・陷阱卡本身的发动。
- `使用1次` 用于效果次数限制。
- 永续魔法・永续陷阱在场上已有的效果，通常仍用 `效果1回合仅可使用1次`。

## 4. 效果句法：条件、费用、对象、处理

先把每个效果拆成四个部分：

```text
①：发动条件，费用，以对象为对象可以发动。效果处理。
```

不是每个效果都有四部分，但顺序不要乱。

### 4.1 发动条件

发动条件描述「什么时候可以发动」。

常见写法：

```text
此卡召唤・特殊召唤的情况下可以发动。
此卡被送至墓地的情况下可以发动。
自己场上有「幽灵宝可梦」怪兽存在的情况下可以发动。
在自己的主要阶段可以发动。
对手发动魔法・陷阱・怪兽的效果时可以发动。
```

可选效果写 `可以发动`。

必发效果不写 `可以发动`：

```text
②：此卡与怪兽进行战斗的伤害计算后发动。将该怪兽破坏。
```

`的情况下` 和 `时` 不要随便替换。一般规则：

- `的情况下可以发动` 更宽松，常用于现代卡文。
- `时可以发动` 常用于需要直接响应某个发动或事件的效果。
- 对连锁响应类效果通常写 `...时可以发动`。

### 4.2 费用

费用在发动时支付，写在 `可以发动` 前。

常见费用：

```text
支付500LP可以发动。
舍弃1张手牌可以发动。
将手牌的此卡送至墓地可以发动。
从墓地将此卡除外可以发动。
将自己场上的1只怪兽解放可以发动。
去除此卡的1个超量素材可以发动。
```

有费用又有对象时，费用在前，对象在后：

```text
舍弃1张手牌，以场上的1张卡为对象可以发动。将该卡破坏。
```

不要把费用写到处理里：

```text
不推荐：①：可以发动。舍弃1张手牌，将场上的1张卡破坏。
推荐：①：舍弃1张手牌，以场上的1张卡为对象可以发动。将该卡破坏。
```

### 4.3 对象

取对象的效果必须在 `可以发动` 前明确写 `以...为对象`。

```text
以场上的1只怪兽为对象可以发动。将该怪兽破坏。
以自己墓地的1只「幽灵宝可梦」怪兽为对象可以发动。将该怪兽特殊召唤。
以对手墓地的最多2张卡为对象可以发动。将该卡除外。
```

处理时用 `该卡`、`该怪兽`、`该对象怪兽` 指回对象。

如果处理时仍要求对象满足条件，可以写得更严格：

```text
以场上的1只表侧表示怪兽为对象可以发动。将该表侧表示怪兽的效果无效。
```

如果是不取对象的选择，把选择写在处理部分：

```text
①：可以发动。挑选对手场上的1张卡破坏。
②：可以发动。将对手场上的怪兽全部破坏。
```

不要把不取对象效果写成取对象。

### 4.4 处理

处理写在 `可以发动。` 后面。

常见处理：

```text
从牌组将1只「幽灵宝可梦」怪兽加入手牌。
从手牌将此卡特殊召唤。
将该卡破坏。
将该怪兽除外。
自己抽1张。
给予对手1000伤害。
此卡的攻击力上升1000。
```

需要持续到回合结束：

```text
此卡的攻击力直至回合结束时为止上升1000。
此回合，自己从额外牌组仅可特殊召唤不死族怪兽。
```

以此效果特殊召唤的后续限制：

```text
以此效果特殊召唤的怪兽的效果无效化。
以此效果特殊召唤的怪兽将在结束阶段破坏。
以此效果特殊召唤的怪兽离开场上的情况下除外。
```

## 5. 发动、适用、不入连锁

不是所有文本都要写 `可以发动`。

### 5.1 入连锁效果

会发动的效果，通常写编号和 `可以发动` 或 `发动`。

```text
①：此卡召唤・特殊召唤的情况下可以发动。从牌组将1张...加入手牌。
```

### 5.2 永续效果

永续效果不写 `可以发动`。

```text
①：只要此卡存在于怪兽区域，自己场上的「幽灵宝可梦」怪兽的攻击力上升500。
②：场上的此卡不会因对手的效果被破坏。
```

### 5.3 召唤规则或特殊召唤手续

不入连锁的特殊召唤通常写 `可从手牌特殊召唤`，不写 `发动`。

```text
①：自己场上有「幽灵宝可梦」怪兽存在的情况下，此卡可从手牌特殊召唤。
```

如果它在脚本中是起动效果或诱发效果，则写 `可以发动。从手牌将此卡特殊召唤。`

卡文必须和脚本实现一致。

## 6. 连接词与处理顺序

复杂处理不要随便用连接词。连接词会影响「是否同时发生」「前一处理失败后是否继续」以及「最后发生了什么」。

简化原则：

- 能拆成两句就拆成两句。
- 必须表达先后关系时，用 `然后`。
- 必须表达同时或附加关系时，先找官方近似卡文，再决定用词。
- 不确定时，不要自己发明 `并且`、`再`、`接着`。

常用写法：

```text
将该卡破坏。然后，自己抽1张。
将该发动无效并破坏。
从牌组将1张...加入手牌。此回合，自己不可...
```

PSCT 中常见英文连接词大意：

- `then`：A 先发生，B 后发生；A 没发生则不继续 B。
- `also`：A 和 B 同时处理，尽量处理能处理的部分。
- `and if you do`：A 和 B 同时看待，但 B 依赖 A。
- `and`：A 和 B 绑定，通常必须都能处理。

中文 DIY 卡文不要强行逐字翻译这些英文词。优先参考官方简中已有类似效果。

## 7. 无效、破坏、连锁限制

### 7.1 无效发动

用于无效「卡或效果的发动」：

```text
对手发动魔法・陷阱・怪兽的效果时可以发动。将该发动无效并破坏。
```

如果不破坏：

```text
将该发动无效。
```

### 7.2 无效效果

用于效果已经发动，只无效效果：

```text
将该效果无效。
```

### 7.3 无效表侧怪兽效果

用于持续性地让场上怪兽无效：

```text
以场上的1只表侧表示怪兽为对象可以发动。将该表侧表示怪兽的效果直至回合结束时为止无效。
```

### 7.4 连锁限制

```text
应对此效果的发动，对手不可发动魔法・陷阱・怪兽的效果。
```

或针对特定类别：

```text
应对此卡的发动，对手不可发动魔法・陷阱卡。
```

不要写成口语：

```text
对手不能连锁这个效果。
```

## 8. 锁定、限制与副作用

发动后的自肃：

```text
此效果发动后，直至回合结束时为止，自己不可从额外牌组特殊召唤不死族以外的怪兽。
```

卡本身发动后的自肃：

```text
此卡发动后，直至回合结束时为止，自己不可特殊召唤「幽灵宝可梦」怪兽以外的怪兽。
```

处理后的回合限制：

```text
此回合，自己不可进行战斗阶段。
此回合，自己仅可用「幽灵宝可梦」怪兽攻击。
```

特殊召唤怪兽限制：

```text
以此效果特殊召唤的怪兽不可攻击。
以此效果特殊召唤的怪兽不可作为连接素材。
以此效果特殊召唤的怪兽离开场上的情况下除外。
```

## 9. 选择效果与项目符号

二选一或多选一效果用 `●`。

```text
①：可以从以下效果中选择1个发动。
●从牌组将1只「幽灵宝可梦」怪兽加入手牌。
●以自己墓地的1只「幽灵宝可梦」怪兽为对象，将该怪兽特殊召唤。
```

如果多个选项各自有次数限制，写清楚：

```text
此卡名的以下效果1回合仅可各选择1次。
①：可以从以下效果中选择1个发动。
●...
●...
```

如果同一回合只能选一个：

```text
此卡名的①效果1回合仅可使用1次。
①：可以从以下效果中选择1个发动。
●...
●...
```

## 10. 常见卡种模板

### 10.1 主卡组怪兽

```text
此卡名的①②效果1回合仅可各使用1次。
①：此卡召唤・特殊召唤的情况下可以发动。从牌组将1张「幽灵宝可梦」魔法・陷阱卡加入手牌。
②：从墓地将此卡除外，以自己墓地的1只「幽灵宝可梦」怪兽为对象可以发动。将该怪兽特殊召唤。
```

### 10.2 融合怪兽

```text
「幽灵宝可梦」怪兽＋暗属性怪兽
此卡名的①②效果1回合仅可各使用1次。
①：此卡融合召唤的情况下可以发动。挑选场上的1张卡破坏。
②：此卡被对手破坏的情况下可以发动。从墓地将1只「幽灵宝可梦」怪兽特殊召唤。
```

### 10.3 同步怪兽

```text
协调＋协调以外的怪兽1只以上
此卡名的①②效果1回合仅可各使用1次。
①：此卡同步召唤的情况下可以发动。从牌组将1只「幽灵宝可梦」怪兽送至墓地。
②：对手发动怪兽的效果时可以发动。将该效果无效。
```

### 10.4 超量怪兽

```text
等级4「幽灵宝可梦」怪兽×2
此卡名的①②效果1回合仅可各使用1次。
①：去除此卡的1个超量素材可以发动。从牌组将1张「幽灵宝可梦」卡加入手牌。
②：此卡被破坏的情况下可以发动。将自己墓地的1只「幽灵宝可梦」怪兽特殊召唤。
```

### 10.5 连接怪兽


## 11. 常见改写示例

### 11.1 重复次数限制

不规范：

```text
①：此卡名的①效果1回合只能使用1次。此卡召唤成功时可以发动...
②：此卡名的②效果1回合只能使用1次。自己场上的这张卡以外的...
```

规范：

```text
此卡名的①②效果1回合仅可各使用1次。
①：此卡召唤成功的情况下可以发动。...
②：...
```

### 11.2 费用和处理混在一起

不规范：

```text
①：可以发动。丢弃1张手牌，破坏场上1张卡。
```

规范：

```text
①：舍弃1张手牌，以场上的1张卡为对象可以发动。将该卡破坏。
```

### 11.3 不取对象写成取对象

如果脚本是处理时选择：

```text
①：此卡特殊召唤的情况下可以发动。挑选对手场上的1张卡破坏。
```

如果脚本是发动时选择对象：

```text
①：此卡特殊召唤的情况下，以对手场上的1张卡为对象可以发动。将该卡破坏。
```

卡文必须和脚本一致。

### 11.4 魔法卡发动限制

不规范：

```text
此卡名的效果1回合只能使用1次。
①：发动后，从牌组检索...
```

规范：

```text
此卡名的卡1回合仅可发动1张。
①：从牌组将1只「幽灵宝可梦」怪兽加入手牌。
```

### 11.5 口语特殊召唤

不规范：

```text
这张卡被送墓时，可以拉墓地一只本家。
```

规范：

```text
①：此卡被送至墓地的情况下，以自己墓地的1只「幽灵宝可梦」怪兽为对象可以发动。将该怪兽特殊召唤。
```

## 12. Agent 写卡文流程

其他 agent 在写入 CDB 前，必须按这个顺序处理：

1. 先用自然语言写清楚每个效果的规则逻辑。
2. 判断每个效果是否入连锁。
3. 判断是否有费用。
4. 判断是否取对象。
5. 判断处理是选择、全部处理，还是引用发动时对象。
6. 判断次数限制是卡名限制、软次数限制，还是魔法・陷阱卡发动限制。
7. 按模板写成卡文。
8. 对照脚本确认卡文和实现一致。
9. 再写入 `texts.desc`。

写入前检查清单：

```text
[ ] 是否统一使用「此卡」「此卡名」「对手」「牌组」？
[ ] 编号效果是否使用 ①：②：③：？
[ ] 共通次数限制是否放在编号效果前？
[ ] 魔法・陷阱卡本身是否写成「此卡名的卡1回合仅可发动1张」？
[ ] 每个发动效果是否能分清条件、费用、对象、处理？
[ ] 取对象效果是否写了「以...为对象可以发动」？
[ ] 不取对象效果是否没有误写对象？
[ ] 费用是否写在「可以发动」前？
[ ] 必发效果是否没有写「可以发动」？
[ ] 永续效果是否没有写成发动效果？
[ ] 自肃和后续限制是否写清持续时间？
[ ] 卡文是否和 Lua 脚本行为一致？
```

## 13. 与 Lua 脚本的对应提醒

卡文不是装饰文本，必须能对应脚本。

常见对应关系：

```text
以...为对象可以发动        -> 脚本里应有 SetTarget / SelectTarget 一类取对象逻辑
挑选...                  -> 多数情况下是处理时选择，不应在发动时取对象
舍弃/支付/解放...可以发动 -> 脚本里应作为 cost
此卡名的①效果...         -> 脚本里应有同名次数限制 count limit
此卡名的卡1回合仅可发动1张 -> 魔法・陷阱的卡发动本身需要限制
只要此卡存在...           -> 通常是永续效果，不应入连锁
```

如果脚本做不到卡文描述的精度，优先改脚本；如果 MDPro3 API 限制导致做不到，必须改卡文，不能让文字承诺不存在的规则。
